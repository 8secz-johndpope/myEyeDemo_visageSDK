61 * The FaceTracker2 example project demonstrates the following capabilities of visage|SDK:
62 *
63 * - real-time head/facial features tracking on multiple faces from video file or camera, based on the VisageSDK::VisageTracker tracker.
64 * - liveness verification from a camera
65 *
66 * <br/><br/>
67 * In order to try the sample application, you need to have a valid license key. More information on how to obtain license key and include it in the application can be found in
68 * <a href="../../../../../../doc/OpenGL/licensing.html"><em>Licensing</em></a> section of the documentation.
69 * <br/><br/>
70 *
71 * For a quick start just run the demo, and select "Track From Camera"; or select "Track From Avi video" from the "Tracking" menu and select one of the video files available in the <a href="../../../../data/Videos">Samples/OpenGL/data/Videos folder</a>
72 * which may require, depending on the system, installation of <a href="http://www.xvidmovies.com/codec/" target="_blank">Xvid video codec</a>.
73 * \n\n For liveness verification demonstration, please select Liveness -> Perform liveness.
74 *
75 * The specific classes and methods that demonstrate visage|SDK are:
76 * - CVisionExampleDoc::OnTrackingTrackfromavivideo(): Performs real time tracking from a video file.
77 * - CVisionExampleDoc::OnTrackingTrackfromcam(): Performs real time tracking from a camera.
78 * - CVisionExampleDoc::OnTrackingTrackinimage(): Performs real time tracking from an image.
79 * - CVisionExampleDoc::OnTrackingStop(): Stops the real time face tracking.
80 * - CVisionExampleDoc::OnLivenessStart() Performs liveness verification from a camera.
81 *
82 * Further classes and methods that are important for detailed understanding how this
83 * sample project is implemented and how it used visage|SDK are:
84 * - VisageSDK::FaceData class that is used for storing all the tracking data
85 *
86 * \section S1_export_to_file Tracking data into a file
87 *
88 * This example project includes code that implements writing of tracking data into a text file. The functionality is disabled by default and can be enabled by choosing Tracking from menu and selecting Write tracking data.
89 * In VisionExampleDoc.cpp close to the top are the lines defining VS_WRITE_AU and VS_WRITE_HEADPOSE. Each of these constants controls the writing of one type of data,
90 * specifically Action Units and head pose. For example, to disable writing of head pose and leave only Action Units, simply comment the line defining VS_WRITE_HEADPOSE.
91 *
92 * When activated, the data is written into a text file with values delimited by colon (;). The first line is the header.
93 *
94 * When tracking from an image, the output data file is created in the same folder with the image and has the same name as the image with extension .txt.
95 *
96 * When tracking from a video file, the output data file is created in the same folder with the video file and has the same name as the video file with extension .txt.
97 *
98 * When tracking from camera, the output file is created in the tracker working folder (by default this is "Visage Technologies\visageSDK\Samples\data").
99 * The name of the file is cam-data-<time>.txt, where "time" is the date and time when data was recorded.
100 */
