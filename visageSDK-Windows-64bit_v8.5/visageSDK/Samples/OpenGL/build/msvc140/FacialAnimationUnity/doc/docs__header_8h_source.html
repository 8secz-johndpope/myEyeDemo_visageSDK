28  * The FacialAnimationUnityDemo sample project demonstrates the integration of visage|SDK with Unity game engine
29  * and is aimed at developers starting to use face tracking functionality of visage|SDK to drive facial animation
30  * using blendshapes and bones in the Unity game engine.
31  *
32  *
33  * \section using Using the sample application
34  *
35  * <br/><br/>
36  * In order to try the sample application, you need to have a valid license key. More information on how to obtain license key and include it in the application can be found in
37  * <a href="../../../../../../doc/OpenGL/licensing.html"><em>Licensing</em></a> section of the documentation.
38  * <br/><br/>
39  *
40  * To start the application, double-click on FacialAnimationUnity.exe located in \if WIN64_DOXY <i>visageSDK/bin64 </i>\else <i>visageSDK/bin </i>\endif folder.
41  *
42  * As soon as the application starts, tracking will begin. When a face is found in the camera frame, the character will
43  * start moving - mimicking the user's facial expressions. Try opening your mouth, smiling or moving your eyebrows. Available buttons are: Play/Pause in the lower left corner, Switch camera in the lower right corner and Normal/Mask in the upper right corner of the screen.
44  * If no license key is present the tracking will not work - please refer to the <a href="../../../../../../doc/OpenGL/licensing.html">licensing section</a> for details.

60  * -# Select and open the "Visage Tracker/Jones/jones" scene.
61  *
62  * \subsection generateApp Building the application
63  *
64  * To build the application follow these steps:
65  * -# Build the application by selecting File->Build settings... and under Platform choose PC and Mac Standalone and set Windows as target platform. Check if the correct Architecture is set: \if WIN64_DOXY <i>x86_64</i>\else <i>x86</i>\endif.
66  * -# Press the Build button to generate the project, a dialog window will appear. In this window, locate and set destination folder to \if WIN64_DOXY <i>visageSDK/bin64</i>\else <i>visageSDK/bin</i>\endif.

104  * \section implementation Implementation overview
105  *
106  * The project consists of a main scene with the Unity’s GameObjects that provide different functionalities. A GameObject
107  * is the main building block of a scene in Unity. It consists of different Components and can parent other GameObjects.
108  * Manipulation of GameObjects is done with Scripts. It is recommended for developers to get familiar with Unity basics
109  * before continuing. The integration with visage|SDK is done through a plugin (VisageTrackerUnityPlugin) that wraps
110  * native code calls to visage|SDK and provides functions that can be called from Unity scripts. The source code for the plugin can be found in the VisageTrackerUnityPlugin project in the samples folder.
111  *
112  *
113  * \subsection trackerGO Tracker GameObject
114  *
115  * The core of the example is the Visage Tracker GameObject. It handles communication with the tracker
116  * (starts it and gets results from it). The behaviour of the script can be modified by changing its properties and the script
117  * code. Other properties can be added as well. The existing Visage Tracker script properties of interest are as follows:<br/>
118  * • Config File Standalone: filename of the tracker configuration file (see the <a href="../../../../../../doc/OpenGL/VisageTracker Configuration Manual.pdf">VisageTracker Configuration manual</a> for information on modifying the tracker configuration) that will be used by the tracker for standalone platforms<br/>
119  * • Config File Editor: filename of the tracker configuration file (see the <a href="../../../../../../doc/OpenGL/VisageTracker Configuration Manual.pdf">VisageTracker Configuration manual</a> for information on modifying the tracker configuration) that will be used by the tracker in the editor<br/>
120  * • BindingConfigurations: list of binding configurations (see the Binding Configuration section on this page) used in the scene<br/>
121  *
122  *
123  * \subsection cameras Cameras
124  *
125  * There are two cameras in the scene, one for displaying the character (Main Camera) and other for the background (Background Camera).
126  * The main camera "Field of View" is automatically set according to the Visage Tracker focus (camera_focus parameter in
127  * configuration file).
128  *
129  *
130  * \subsection jonesObject Jones object
131  *
132  * Jones is the character with defined blendshape animations which are controlled using the a binding configuration and bones
133  * which are controlled using the <i>Visage Tracker/Jones/Scripts/Movement.cs</i> script. The visage|SDK provides relevant information about facial expressions through action units. Their values are then mapped to specific blendshapes using
134  * binding configurations in the <i>VisageTracker.cs</i> script. For more information on creating characters with such blendshapes, consult the <a href="../../../../../../doc/OpenGL/modeling_guide.pdf" target="_blank">Modeling Guide</a>.
135  *
136  *
137  * \subsection bindingConfiguration Binding configuration
138  *
139  * Binding configurations map action unit values to specific blendshapes and animate them. A reference binding configuration used to animate Jones is located in <i>Visage Tracker/Jones/jones.bind.txt</i>. <br/>
140  * <i>ActionUnitBinding.cs</i> components are created based on the <i>VisageTracker.cs</i> Binding Configuration list in the current scene at runtime. They receive action unit data from the Visage Tracker component and map them to [0,1] values based on the info given in the binding configuration.
141  * You can omit the binding configuration and create the <i>ActionUnitBinding.cs</i> components manually for more control.
142  *
143  *
144  * \subsection plugin VisageTrackerUnityPlugin
145  * The integration of visage|SDK face tracking with Unity is done through a plugin that wraps native code calls and
146  * provides functions that can be called from Unity scripts. For more information about the plugin including it's source code, see the VisageTrackerUnityPlugin project and its documentation.
147  *
