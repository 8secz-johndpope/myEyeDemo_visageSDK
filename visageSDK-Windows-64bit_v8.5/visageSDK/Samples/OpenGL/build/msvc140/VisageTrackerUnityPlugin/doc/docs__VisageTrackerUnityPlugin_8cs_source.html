7 /// <summary>
8 /// Provides high-level functions for using visage|SDK face tracking capabilities in Unity game engine.
9 ///Additionally, provides high-level functions for camera management such as _openCamera(), _closeCamera() and _getCameraInfo().
10 ///
11 ///It implements high-level wrapper functions around key functionalities from visage|SDK that enable usage of visage|SDK face tracking capabilities in Unity C# scripts.
12 ///
13 ///VisageTracker is a face tracker capable of tracking the head pose and facial features for multiple faces in video coming from a video file, camera or other sources.
14 ///Maximum number of faces that can be tracked is limited to 20 within native plugin library with MAX_FACES parameter.
15 ///Parametar faceIndex with value between 0 and MAX_FACES is passed to the functions within native plugin library to obtain the data for a particular tracked face.
16 ///All functions that return the data obtained from the tracker, internally check the tracker status as well as the value of the parameter faceIndex.
17 ///
18 ///The tracker is fully configurable through comprehensive tracker configuration files.visage|SDK contains optimal configurations for common uses such as head tracking and facial features tracking.Please refer to the VisageTracker Configuration Manual for the list of available configurations.
19 ///The VisageTracker Configuration Manual provides full detail on all available configuration options, allowing to customize the tracker in terms of performance, quality, and other options.
20 ///
21 ///High-level functions offer the following outputs:
22 /// - 3D head pose
23 /// - facial expression
24 /// - gaze direction information
25 /// - eye closure
26 /// - facial feature points
27 /// - full 3D face model, textured
28 ///
29 ///The tracker can apply a smoothing filter to tracking results to reduce the inevitable tracking noise. Smoothing factors are adjusted separately for different parts of the face.The smoothing settings in the supplied tracker configurations are adjusted conservatively to avoid delay in tracking response, yet provide reasonable smoothing. For further details please see the smoothing_factors parameter array in the VisageTracker Configuration Manual.
30 ///
31 ///Frames are fetched natively using VideoInput library by calling _grabFrame() function and are applied to the Unity-created texture using OpenGL in _setFrameData() function.
32 ///Function _grabFrame() needs to be called before each sequential call to the _track() function to obtain a new frame.
33 ///
34 ///Afterwords, high-level function such as _getFeaturePoints2D() are called to obtain tracker results for the given frame.
35 ///
36 ///The tracker requires a set of data and configuration files, available in Samples/data folder.
37 ///For most Unity applications usage of Application.streamingAssetsPath folder is encouraged.
38 ///
39 /// </summary>
40 ///

63 * {
64 * // create texture with dimensions that are dependent on image dimensions as the nearest heigher power of number 2

74 * // send memory address of textures' pixel data to VisageTrackerUnityPlugin
75 * VisageTrackerNative._setFrameData(texturePixelsHandle.AddrOfPinnedObject());

545 /**
546 * Returns the feature points position in normalized 2D screen coordinates, indicator whether the point is defined, indicator whether the point is detected or estimated
547 * and feature point quality for the given faceIndex.
548 *
549 * The 2D feature point coordinates are normalised to image size so that the lower left corner of the image has coordinates 0,0 and upper right corner 1,1.
550 *
551 * Function returns position for N feature points in the N*2-dimensional <i>positions</i> array, an indication of whether the point is defined as 0 and 1 in the N-dimensional <i>defined</i> array and detected as 0 and 1 in the N-dimensional <i>detected</i> array.
552 * The quality of each point as value between 0 and 1 is returned in <i>quality</i> array.
553 * For each point, its group and index are specifically listed in the <i>groups</i> and <i>indices</i> arrays.
554 *
555 * An example of use:
556 * \code
557 * // an example for points 2.1, 8.3 and 8.4
558 * const int N;
559 * int[] groups = new int[N] {2, 8, 8};
560 * int[] indices = new int[N]{ 1, 3, 4 };
561 * float[] positions2D = new float[2*N];
562 * int[] defined = new int[N];
563 * int[] detected = new int[N];
