Detailed Description

VisageLivenessBlink is a class that detects whether the person on the live video stream has blinked.

In order for action to be detected accurately, it is required that the face is frontal for 30 frames before performing blink.
During blink verification, face data from latest 30 frames ("face data sequence"), i.e. last 30 calls of update() is used to verify head stability (STATE_WAIT_FOR_STABLE). Each time the action is reset, face data sequence will be cleared and the action will need to go through initializing state (STATE_ACTION_INITIALIZING) to store initial face data sequence.

Action should be updated for each frame in order to update its internal state. The action can be in one of the following states (states are sequentially verified in each call of update()):

  1. STATE_WAIT_FOR_FRONTAL Face is not frontal. Action is reset.
  2. STATE_ACTION_INITIALIZING Action is storing first set of face data sequence. After each reset, action needs to reinitialize.
  3. STATE_WAIT_FOR_STABLE Head is not stable. In order for action to be verified successfully, the user should keep his head still.
  4. STATE_WAIT_FOR_ACTION Data is processed in order to verify blink.
  5. STATE_ACTION_VERIFIED Action is verified.
